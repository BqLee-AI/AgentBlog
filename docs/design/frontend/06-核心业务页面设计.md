# 06 · 核心业务页面设计

> 本篇逐域拆解**阅读端**与**写作后台**的所有页面：页面构成、对接的后端接口（后端 [04~11](../README.md)）、关键交互、特殊约束。
> 表单/数据获取/组件的通用范式见 [03](./03-API-层与请求封装.md)、[04](./04-状态管理与数据获取.md)、[05](./05-表单校验与组件体系.md)。

## 一、页面总览

### 阅读端（公开）

| 页面 | 路由 | 后端接口 | 说明 |
|------|------|----------|------|
| 首页 | `/` | `GET /api/posts?status=published` | 最新文章列表 |
| 文章列表 | `/posts` | `GET /api/posts`（带 tag/page） | 标签筛选 + 分页 |
| 文章详情 | `/posts/:slug` | `GET /api/posts/:slug` | Markdown 渲染 + 作者归属 |

### 写作后台（JWT + 守卫）

| 页面 | 路由 | 后端接口 | 守卫 |
|------|------|----------|------|
| 后台框架 | `/admin` | — | RequireAuth |
| 文章管理 | `/admin/posts` | `GET /api/posts`（含草稿） | RequireAuth |
| 文章编辑 | `/admin/posts/new`、`/admin/posts/:id/edit` | `GET by-id`、`POST/PATCH /api/posts` | RequireAuth |
| 标签管理 | `/admin/tags` | `GET/POST/DELETE /api/tags` | RequireRole(admin+) |
| 用户管理 | `/admin/users` | `GET /api/users`、`POST /api/auth/register`、`PATCH /api/users/:id/role` | RequireRole(admin+) |
| 额度流水 | `/admin/credits` | `GET /api/credits/me/logs`、`POST /api/credits/recharge`(admin+) | RequireAuth |
| 我的 Agent | `/admin/agent` | `GET/POST/PATCH/DELETE /api/agents` | RequireAuth |
| API Key | `/admin/agent/keys` | `GET/POST /api/agents/:id/api-keys`、`DELETE /api/api-keys/:id` | RequireAuth |
| 在线对话 | `/chat` | `POST /api/chat`（流式） | RequireAuth（见 [07](./07-在线对话与流式实现.md)） |

## 二、阅读端

> 📌 阅读端页面与后台页面不共用同一套重视觉语气。阅读端优先遵循 **Sophisticated Editorial + Subtle Neo-Futurism**：强调内容层级、留白、阅读节奏与轻量品牌识别；后台继续优先任务效率与高信息密度。

### 2.1 文章列表页（`pages/public/post-list.page.tsx`）

```
┌─────────────────────────────────────────┐
│  标签筛选：[全部] [技术] [随笔] [教程]    │   ← GET /api/tags
├─────────────────────────────────────────┤
│  ┌─────┐ 标题                            │
│  │封面 │ 摘要…        作者 · 时间         │   ← GET /api/posts?tag=&page=
│  └─────┘ [技术] [随笔]                    │
│  ─────────────────────────────────       │
│  ┌─────┐ ……                              │
├─────────────────────────────────────────┤
│            < 1 2 3 4 >                   │   ← 分页
└─────────────────────────────────────────┘
```

要点：
- 标签列表 `useTags()`（`queryFn: tagsApi.list()`），点击切换 `tag` query 参数。
- 文章列表 `usePostList({ page, tag, status: 'published' })`，`placeholderData: prev` 翻页不闪。
- 列表项 `<Link to={`/posts/${post.slug}`}>`，slug 永久稳定（后端 [06 §五](../06-文章模块.md) 铁律），可安心外链。
- 草稿不出现（公开接口后端强制 published）。

### 2.2 文章详情页（`pages/public/post-detail.page.tsx`）

```tsx
const { slug } = useParams()
const { data: post, isLoading, error } = usePost(slug!)

if (isLoading) return <ListSkeleton rows={3} />
if (error?.isNotFound) return <Navigate to="/not-found" replace />   // 草稿/不存在 → 404
if (error) return <ErrorState message={error.message} onRetry={refetch} />

return (
  <article className="prose max-w-3xl mx-auto">
    <h1>{post.title}</h1>
    <AuthorBadge author={post.author} />
    {post.coverUrl && <img src={post.coverUrl} alt={post.title} className="rounded-md" />}
    <Markdown content={post.content} />
    <div className="flex gap-2">
      {post.tags.map((t) => <Badge key={t.id}><Link to={`/posts?tag=${t.slug}`}>{t.name}</Link></Badge>)}
    </div>
  </article>
)
```

> 💡 阅读端是公开接口，`usePost` 的 queryFn 走 `public: true`（不注入 Bearer），见 [03 §六](./03-API-层与请求封装.md)。

## 四、文章管理（后台）

### 4.1 文章列表（`pages/admin/posts.page.tsx`）

与阅读端列表类似，但：
- 含草稿：`usePostListAdmin({ status?: 'draft'|'published', authorId? })`，**带 Bearer**。
- 每行有「编辑」「删除」操作。编辑/删除按钮按资源归属显隐（见 [02 §九](./02-路由设计与权限守卫.md)）。
- 「新建文章」按钮跳 `/admin/posts/new`。

### 4.2 文章编辑（`pages/post-edit.page.tsx`）

新建与编辑共用一页（按有无 `:id` 区分）。核心字段 + 上传封面。

```
┌──────────────────────────────────────────┐
│ 标题：[__________________]                │
│ 摘要：[__________________]                │
│ 封面：[上传图片]  [预览图]                 │   ← POST /api/upload → PATCH coverUrl
│ 标签：[多选：技术 随笔 教程]               │   ← GET /api/tags
│ 正文：[Markdown textarea]    [预览]        │
│ 状态：○ 草稿  ● 已发布                    │
│                            [取消] [保存]   │
└──────────────────────────────────────────┘
```

```tsx
const { id } = useParams()
const isEdit = !!id
const { data: post } = usePostForEdit(Number(id))   // 仅编辑时取
const form = useForm<CreatePostDTO>({ resolver: zodResolver(createPostSchema), values: post })

const create = useCreatePost()
const update = useUpdatePost()

const onSubmit = async (dto: CreatePostDTO) => {
  try {
    if (isEdit) await update.mutateAsync({ id: Number(id), dto })
    else await create.mutateAsync(dto)
    navigate('/admin/posts')
  } catch (err) {
    setServerErrors(form, err)
  }
}
```

要点：
- **slug 字段不出现**在表单（后端 [06 §五](../06-文章模块.md) slug 永不可变，用户无需也不应编辑）。
- 封面上传：先 `uploadApi.uploadImage(file, 'cover')` 拿 url，`form.setValue('coverUrl', url)`。上传中显示进度/禁用保存。
- `tagIds` 多选，选项来自 `GET /api/tags`。

## 五、图片上传（复用组件）

封面上传、用户/Agent 头像上传共用 `<ImageUpload>` 组件，内部调 `uploadApi.uploadImage`：

```tsx
// components/image-upload.tsx
export function ImageUpload({
  purpose,
  value,
  onChange,
}: {
  purpose: 'cover' | 'avatar'
  value?: string
  onChange: (url: string) => void
}) {
  const [uploading, setUploading] = useState(false)
  const inputRef = useRef<HTMLInputElement>(null)

  const handleFile = async (file: File) => {
    setUploading(true)
    try {
      const result = await uploadApi.uploadImage(file, purpose)
      onChange(result.url)
      toast.success('上传成功')
    } catch (err) {
      // 上传失败（大小/类型）走全局 toast（message 已中文）
    } finally {
      setUploading(false)
    }
  }

  return (
    <div>
      {value && <img src={value} alt="预览" className="w-32 h-32 object-cover rounded" />}
      <input
        ref={inputRef}
        type="file"
        accept="image/jpeg,image/png,image/webp,image/gif"   // 与后端 [11 §三] ALLOWED 对齐
        className="hidden"
        onChange={(e) => e.target.files?.[0] && handleFile(e.target.files[0])}
      />
      <Button type="button" variant="outline" disabled={uploading} onClick={() => inputRef.current?.click()}>
        {uploading ? '上传中…' : value ? '更换图片' : '上传图片'}
      </Button>
    </div>
  )
}
```

> 💡 accept 属性对应后端 [11 §三](../11-图片上传与存储.md) 允许的 MIME。前端先挡（少一次网络请求），后端再挡（安全兜底）。大小限制（5MB）后端会校验，前端可加 `file.size > 5*1024*1024` 提前提示。

```
当前额度：5000 credits                                       ← 来自 useAuth().user.credits
┌──────────────────────────────────────────────────────────┐
│ 时间          变动    类型        原因                      │
├──────────────────────────────────────────────────────────┤
```

## 十、API Key（`pages/admin/api-keys.page.tsx`）

⚠️ 本页最关键的 UX：**明文 Key 仅签发时显示一次**（后端 [07 §2.1](../07-Agent-与-API-Key-模块.md)）。

```
┌──────────────────────────────────────────────────────┐
│ 我的 Agent 的 API Key                                  │
│                                  [签发新 Key]          │
├──────────────────────────────────────────────────────┤
│ 名称            前缀            状态    操作            │
│ 我的本地客户端   sk_live_3f9a…   启用    [吊销]         │
│ (未命名)        sk_live_b7c1…   已吊销  —              │   ← 已吊销不可再吊销
└──────────────────────────────────────────────────────┘

签发后弹窗（一次性！）：
┌──────────────────────────────────────────────────────┐
│ ⚠️ 请立即保存，关闭后无法找回                          │
│                                                       │
│ sk_live_3f9a2b1c...48hex            [复制]            │
│                                                       │
│ 配置示例（Claude Desktop）：                           │
│ { "mcpServers": { "agentblog": {                      │
│     "url": "https://你的域名/mcp",                     │
│     "headers": { "X-API-Key": "sk_live_..." }         │
│ }} }                                                  │
│                                                       │
│              [我已保存，关闭]                           │
└──────────────────────────────────────────────────────┘
```

```tsx
// features/api-key/api-key-reveal-dialog.tsx（关键交互）
const issue = useIssueApiKey()
const [revealedKey, setRevealedKey] = useState<string | null>(null)
const { copy } = useCopyToClipboard()

const handleIssue = async () => {
  const result = await issue.mutateAsync({ agentId, name })
  setRevealedKey(result.key)   // 明文仅在此时持有，关闭即丢
}

const handleClose = () => {
  setRevealedKey(null)         // 确保组件卸载/关闭后内存不留明文
  setOpen(false)
}
```

要点：
- 列表 `useApiKeys(agentId)`（`GET /api/agents/:id/api-keys`），只返回 prefix（无明文）。
- 签发 `useIssueApiKey`，返回明文，**立刻**让用户复制 + 显示醒目警告。
- 关闭弹窗后 `setRevealedKey(null)`，内存不留明文（虽然 sessionStorage 也不存，但这是 UX 防误关）。
- 提供 MCP 客户端配置示例（后端 [09 §六](../09-MCP-Server-实现.md) 的 JSON）。
- 吊销用 `<IfRole>` 或状态判断：已吊销的 Key 不显示吊销按钮。

> 📌 **UX 铁律**：签发后必须强提示"仅此一次"。用户没保存就关闭 = 永久丢失（只能再签发新 Key）。文案用警示色（黄/红）+ 图标。

## 十一、页面间联动

| 场景 | 处理 |
|------|------|
| 对话扣费后顶栏 credits 变化 | 对话 `onFinish` 后 `qc.invalidateQueries({ queryKey: queryKeys.me })` |
| 充值后流水列表更新 | `qc.invalidateQueries({ queryKey: queryKeys.credits.myLogs })` + 刷新 me |
| 删除 Agent 后跳转 | 成功后 `navigate('/admin/agent')`，因 Agent 已不存在，页面回退到创建态 |
| 文章保存后列表含新文章 | `invalidateQueries({ queryKey: queryKeys.posts.all })` |
