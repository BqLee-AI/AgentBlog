## 一、为什么需要这套文档

后端文档（00~14）已完整定义 API 契约、鉴权、计费、MCP、流式协议，但**前端架构此前缺失**。本文档填补这一空白，目标是：

1. 让前端同学（黄宇超 / 李瑞东等）拿到就能开干，无需反向猜测后端约定。
2. 把后端在 `@agentblog/shared` 暴露的类型 / Zod schema / 错误码**落到前端的每一层**，杜绝契约漂移。
3. 沉淀统一的数据获取、鉴权、表单、流式、错误处理范式，多人协作不各写一套。

## 四、文档约定

- 代码示例运行在 **Vite + React 18 + TS（strict）**，与根 `tsconfig.base.json` 一致。
- 路径前缀统一为 `apps/web/`，下文目录树省略该前缀（与后端文档 `apps/api/` 约定对称）。
- **API 契约单一真相源**：跨前后端复用的类型、Zod schema、错误码、枚举**统一从 `@agentblog/shared` 引入**，前端禁止重写或手抄同名类型（见后端 [02 §1.1](../02-目录结构与分层规范.md)）。
- 所有 API 响应遵循后端 [12 · 统一错误处理与响应规范](../12-统一错误处理与响应规范.md) 的 `{ ok, data, error }` 结构。
- 📌 标记表示"需求文档 / 后端文档强约束"，不可随意变更。
- 💡 标记表示"实现建议"，可按团队习惯调整。
