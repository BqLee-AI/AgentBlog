# 02 · 路由设计与权限守卫

> 本篇定义前端**路由树**、**懒加载策略**、**公开/受保护路由分区**，以及对应后端 [05 RBAC](../05-RBAC-权限控制.md) 的**前端守卫体系**。
> 用 React Router 6 的 data router（`createBrowserRouter`）。

---

## 一、路由总览

前端按后端 [00 §二](../00-技术选型与架构总览.md) 的三大功能域组织路由：

| 区域 | 路径 | 鉴权 | 守卫 | 说明 |
|------|------|------|------|------|
| 阅读端（公开） | `/`、`/posts`、`/posts/:slug` | 无 | 无 | 公开已发布文章 |
| 登录 | `/login` | 无 | 仅未登录可访问 | 登录后跳回 `from` |
| 在线对话 | `/chat` | JWT | `RequireAuth` | 消耗自己 credits |
| 写作后台 | `/admin/*` | JWT | `RequireAuth` | 所有登录用户可进框架 |
| └ 文章 | `/admin/posts`、`/admin/posts/new`、`/admin/posts/:id/edit` | JWT | `RequireAuth` | 自己/admin+ 可编辑 |
| └ 标签 | `/admin/tags` | JWT | `RequireRole('admin','super_admin')` | admin+ 建删 |
| └ 用户 | `/admin/users` | JWT | `RequireRole('admin','super_admin')` | admin+ 管理 |
| └ Credits | `/admin/credits` | JWT | `RequireAuth`（管理员功能按按钮级控制） | 自己看流水；admin+ 充值 |
| └ Agent | `/admin/agent` | JWT | `RequireAuth` | 每用户≤1 |
| └ API Key | `/admin/agent/keys` | JWT | `RequireAuth` | 管理自己 Agent 的 Key |
| 403 | `/forbidden` | 无 | — | 资源/角色不足兜底 |
| 404 | `*` | 无 | — | 兜底 |

> 📌 角色矩阵严格对齐后端 [05 §一](../05-RBAC-权限控制.md)。前端守卫只做"早拦截、好体验"，**最终权限以后端为准**——前端放行的请求后端仍会校验，前端拦截的请求后端也会拒绝。

## 二、路由树（`src/app/router.tsx`）

用 `createBrowserRouter`，配合 `lazy()` 做路由级代码分割（每个页面独立 chunk）。

```tsx
import { createBrowserRouter, Outlet, lazy } from 'react-router-dom'
import { RequireAuth } from '@/features/auth/require-auth'
import { RequireRole } from '@/features/auth/require-role'
import { SiteHeader } from '@/components/layout/site-header'

// 路由级懒加载：每个 page 独立 chunk，首屏只加载阅读端
const lazyPage = (loader: () => Promise<{ default: React.ComponentType }>) =>
  lazy(loader)

const Home = lazyPage(() => import('@/pages/public/home.page'))
const PostList = lazyPage(() => import('@/pages/public/post-list.page'))
const PostDetail = lazyPage(() => import('@/pages/public/post-detail.page'))
const Login = lazyPage(() => import('@/pages/login.page'))
const Chat = lazyPage(() => import('@/pages/chat.page'))
const AdminLayout = lazyPage(() => import('@/pages/admin/layout.page'))
const AdminPosts = lazyPage(() => import('@/pages/admin/posts.page'))
const PostEdit = lazyPage(() => import('@/pages/admin/post-edit.page'))
const AdminTags = lazyPage(() => import('@/pages/admin/tags.page'))
const AdminUsers = lazyPage(() => import('@/pages/admin/users.page'))
const AdminCredits = lazyPage(() => import('@/pages/admin/credits.page'))
const AdminAgent = lazyPage(() => import('@/pages/admin/agent.page'))
const AdminApiKeys = lazyPage(() => import('@/pages/admin/api-keys.page'))
const NotFound = lazyPage(() => import('@/pages/not-found.page'))
const Forbidden = lazyPage(() => import('@/pages/forbidden.page'))

function PublicLayout() {
  return (
    <div className="flex min-h-screen flex-col">
      <SiteHeader />
      <main className="flex-1">
        <Outlet />
      </main>
    </div>
  )
}

export const router = createBrowserRouter([
  {
    path: '/',
    element: <PublicLayout />,         // 公开域：阅读端 / 登录 / forbidden
    children: [
      { index: true, element: <Home /> },
      { path: 'posts', element: <PostList /> },
      { path: 'posts/:slug', element: <PostDetail /> },
      { path: 'login', element: <Login /> },
      { path: 'forbidden', element: <Forbidden /> },

      // 受保护：对话
      {
        path: 'chat',
        element: <RequireAuth><Chat /></RequireAuth>,
      },
    ],
  },
  {
    path: '/admin',
    element: (
      <RequireAuth>
        <AdminLayout />
      </RequireAuth>
    ),
    children: [
      { index: true, element: <AdminPosts /> },
      { path: 'posts', element: <AdminPosts /> },
      { path: 'posts/new', element: <PostEdit /> },
      { path: 'posts/:id/edit', element: <PostEdit /> },
      {
        path: 'tags',
        element: <RequireRole roles={['admin', 'super_admin']}><AdminTags /></RequireRole>,
      },
      {
        path: 'users',
        element: <RequireRole roles={['admin', 'super_admin']}><AdminUsers /></RequireRole>,
      },
      { path: 'credits', element: <AdminCredits /> },
      { path: 'agent', element: <AdminAgent /> },
      { path: 'agent/keys', element: <AdminApiKeys /> },
      { path: '*', element: <NotFound /> },
    ],
  },
  { path: '*', element: <NotFound /> },
])
```

> 💡 `lazy()` + 动态 `import()` 让 Vite 自动代码分割。公开域与 `/admin` 拆成并列 route branch，避免后台继续继承公开 `SiteHeader`。

`App.tsx`：

```tsx
import { RouterProvider } from 'react-router-dom'
import { RootProviders } from '@/app/root-providers'
import { router } from '@/app/router'

export function App() {
  return (
    <RootProviders>
      <RouterProvider router={router} />
    </RootProviders>
  )
}
```

## 三、公开路由 vs 受保护路由

| 维度 | 公开路由 | 受保护路由 |
|------|----------|------------|
| 鉴权 | 不带 Bearer | 带 Bearer（`request()` 自动注入，见 [03](./03-API-层与请求封装.md)） |
| 守卫 | 无 | `RequireAuth` / `RequireRole` |
| 典型 | 阅读端列表/详情 | 后台、对话 |
| 数据 | 公开接口（`GET /api/posts`，后端强制 published） | 受保护接口（`authMiddleware`） |

> 💡 即使是登录用户访问阅读端，也**不强制**带 token——后端阅读端接口是公开的。若想"登录可看更多"，后端有 `maybeAuth`（见后端 [04 §八](../04-用户认证与会话.md)），前端有 token 就带上即可。

## 四、`RequireAuth` 守卫（登录校验）

```tsx
// src/features/auth/require-auth.tsx
import { Navigate, useLocation } from 'react-router-dom'
import { useAuth } from '@/features/auth/use-auth'
import { authStore } from '@/lib/auth-store'
import { Spin } from '@/components/feedback/spin'
import { ErrorState } from '@/components/feedback/error-state'

export function RequireAuth({ children }: { children: React.ReactNode }) {
  const { status } = useAuth()
  const location = useLocation()

  // 首次加载或显式重试时，先等待 /auth/me 校验（见 04）
  if (status === 'loading' || status === 'idle') {
    return <Spin fullscreen tip="正在验证登录…" />
  }

  // 非 401 的瞬时故障：保留 token，不伪装成未登录，给用户重试入口
  if (status === 'error') {
    return (
      <ErrorState
        className="min-h-screen"
        message="登录状态校验失败，请重试"
        onRetry={() => authStore.setState({ status: 'idle' })}
      />
    )
  }

  // 未登录（无 token，或 401 已被 request 层清态）→ 跳登录页，记住来源
  if (status !== 'authenticated') {
    return <Navigate to="/login" replace state={{ from: location }} />
  }

  return <>{children}</>
}
```

登录页登录成功后，读 `location.state.from` 跳回原页面：

```tsx
// pages/login.page.tsx（节选）
const navigate = useNavigate()
const location = useLocation()
const token = useAuthStore((s) => s.token)
const status = useAuthStore((s) => s.status)

if (status === 'authenticated') {
  const from = (location.state as { from?: Location })?.from?.pathname ?? '/admin'
  return <Navigate to={from} replace />
}

if (token && (status === 'idle' || status === 'loading')) {
  return <Spin fullscreen tip="正在验证登录…" />
}
```

## 五、`RequireRole` 守卫（角色校验）

对应后端 [05 §三](../05-RBAC-权限控制.md) 的 `requireRole` 中间件。前端做"早拦截"，避免用户点进去再被 403。

```tsx
// src/features/auth/require-role.tsx
import { Navigate } from 'react-router-dom'
import { useAuth } from '@/features/auth/use-auth'
import type { Role } from '@agentblog/shared'
import { Forbidden } from '@/pages/forbidden.page'   // 或直接 <Navigate to="/forbidden" />

interface Props {
  roles: Role[]            // 例：['admin', 'super_admin']
  children: React.ReactNode
}

export function RequireRole({ roles, children }: Props) {
  const { user } = useAuth()

  // 理论上 RequireAuth 已挡，这里防御性判断
  if (!user) return <Navigate to="/login" replace />

  if (!roles.includes(user.role)) {
    return <Navigate to="/forbidden" replace />
  }

  return <>{children}</>
}
```

> 💡 角色枚举 `Role` 来自 `@agentblog/shared`，与后端 `user.role` 取值（`super_admin` / `admin` / `user`）完全一致。

## 六、懒加载与 Suspense 配套

`lazy()` 返回的组件首次渲染会触发 chunk 加载，需要 `<Suspense>` 包裹。在 `RootProviders` 全局兜底：

```tsx
// src/app/root-providers.tsx
import { Suspense, useEffect } from 'react'
import { QueryClientProvider } from '@tanstack/react-query'
import { Toaster, toast } from 'sonner'
import { ErrorBoundary } from '@/app/error-boundary'
import { queryClient } from '@/lib/query-client'
import { AuthProvider } from '@/features/auth/use-auth'
import { setUnauthorizedHandler, INSUFFICIENT_CREDITS_EVENT } from '@/lib/request'
import { router } from '@/app/router'
import { PageSkeleton } from '@/components/feedback/page-skeleton'

export function RootProviders({ children }: { children: React.ReactNode }) {
  useEffect(() => {
    setUnauthorizedHandler(() => {
      const current = router.state.location
      if (current.pathname !== '/login') {
        router.navigate('/login', { replace: true, state: { from: current } })
      }
    })
  }, [])

  useEffect(() => {
    const handler = () => toast.error('额度不足，请联系管理员充值')
    window.addEventListener(INSUFFICIENT_CREDITS_EVENT, handler)
    return () => window.removeEventListener(INSUFFICIENT_CREDITS_EVENT, handler)
  }, [])

  return (
    <ErrorBoundary>
      <QueryClientProvider client={queryClient}>
        <AuthProvider>
          <Suspense fallback={<PageSkeleton />}>{children}</Suspense>
          <Toaster />
        </AuthProvider>
      </QueryClientProvider>
    </ErrorBoundary>
  )
}
```

> 💡 `PageSkeleton` 用 shadcn 的 `Skeleton`，按页面大致结构（顶栏+列表/正文）画灰块，体验优于空白。

## 七、路由跳转规范

- **优先声明式**：`<Link to="/posts/xxx">`、`<NavLink>`（带 active 态）。
- **命令式**：`useNavigate()` 用于表单提交后跳转、守卫重定向。
- **带状态的跳转**：登录回跳用 `location.state.from`；列表→编辑带 `state: { from: listPage }` 便于取消返回。

```tsx
// 编辑文章保存后跳回列表
const navigate = useNavigate()
const onSuccess = () => navigate('/admin/posts')

// 列表「新建」按钮
<Link to="/admin/posts/new">
  <Button>新建文章</Button>
</Link>
```

## 八、按角色显示/隐藏导航项（菜单级权限）

后台侧栏菜单按角色渲染，与守卫呼应：

```tsx
// src/components/layout/admin-sidebar.tsx（节选）
import type { Role } from '@agentblog/shared'
import { useAuth } from '@/features/auth/use-auth'

interface NavItem {
  to: string
  label: string
  roles?: Role[]   // 不填 = 所有登录用户可见
}

const NAV: NavItem[] = [
  { to: '/admin/posts', label: '文章' },
  { to: '/admin/agent', label: '我的 Agent' },
  { to: '/admin/agent/keys', label: 'API Key' },
  { to: '/admin/credits', label: '额度流水' },
  { to: '/admin/tags', label: '标签', roles: ['admin', 'super_admin'] },
  { to: '/admin/users', label: '用户管理', roles: ['admin', 'super_admin'] },
]

export function AdminSidebar() {
  const { user } = useAuth()
  const items = NAV.filter((i) => !i.roles || (user && i.roles.includes(user.role)))
  return (
    <nav>
      {items.map((i) => (
        <NavLink key={i.to} to={i.to}>{i.label}</NavLink>
      ))}
    </nav>
  )
}
```

> 📌 **菜单隐藏 ≠ 权限安全**：用户手敲 URL 仍会进入路由。`RequireRole` 守卫是第二道防线，后端 RBAC 是最终防线。三道防线缺一不可。

## 九、按钮级权限（资源归属）

后端 [05 §四](../05-RBAC-权限控制.md) 的资源归属校验（user 只能改自己的文章）前端无法 100% 预判（不知道资源归属），策略：

- **角色级**（admin+ 才显示）：用 `RequireRole` 思路包一个 `<IfRole>` 组件。
- **资源级**（自己的才显示编辑/删除按钮）：拉到数据后比对 `post.authorType==='user' && post.authorId===user.id`。

```tsx
// 角色级
export function IfRole({ roles, children }: { roles: Role[]; children: React.ReactNode }) {
  const { user } = useAuth()
  if (!user || !roles.includes(user.role)) return null
  return <>{children}</>
}

// 用法：只有 admin+ 看到「充值」按钮
<IfRole roles={['admin', 'super_admin']}>
  <Button onClick={openRecharge}>充值</Button>
</IfRole>
```

```tsx
// 资源级（文章编辑/删除按钮）
const { user } = useAuth()
const canEdit =
  post &&
  user &&
  (post.authorType === 'user' && post.authorId === user.id
    ? true
    : user.role === 'admin' || user.role === 'super_admin')

{canEdit && <Button>编辑</Button>}
```

> 💡 与后端 [05 §四](../05-RBAC-权限控制.md) 的判断逻辑一致：自己 或 admin+ 可编辑。

## 十一、与后端鉴权状态的同步

| 后端返回 | 前端处理 |
|----------|----------|
| 正常请求（200/201） | 正常流转 |
| 401 `UNAUTHORIZED` | `request()` 拦截 → 清 token → 跳 `/login`（见 [03](./03-API-层与请求封装.md)） |
| `/auth/me` 的 500 / 网络错误 | 保留 token，`RequireAuth` 进入可重试错误态，不伪装成“未登录” |
| 403 `FORBIDDEN` | toast 显示后端 message；或路由跳 `/forbidden` |
| 402 `INSUFFICIENT_CREDITS` | toast「额度不足，请联系管理员充值」（见 [08](./08-错误处理与用户体验.md)） |
| token 即将过期 | v1 不做主动续期，过期后 401 重登（与后端 [04 §三](../04-用户认证与会话.md) 决策一致） |
