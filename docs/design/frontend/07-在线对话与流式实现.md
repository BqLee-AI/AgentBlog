# 07 · 在线对话与流式实现

> 本篇实现需求 §2.2.3 形态 A：基于 Vercel AI SDK `useChat` 的流式对话，对接后端 [10 · 在线 Agent 运行环境](../10-在线-Agent-运行环境.md)。
>
> **强约束**：流式（UI Message 协议，后端 [10 §三](../10-在线-Agent-运行环境.md)）；**不持久化聊天记录**（需求 §1.2/§5）；按 Token 计费（后端扣，前端刷新余额）。

---

## 一、与后端的协议对齐

后端 `POST /api/chat`（后端 [10 §五](../10-在线-Agent-运行环境.md)）：

- 请求体：`{ messages: UIMessage[] }`（AI SDK 的消息结构）。
- 响应：SSE 流（`toUIMessageStreamResponse`），`useChat` 直接消费。
- 鉴权：`Authorization: Bearer <token>`。
- 前置校验（流开始前返回 HTTP 状态码）：
  - 401：未登录/token 失效。
  - 402：余额不足（后端预检 `credits < 1`）。
  - 400：未创建 Agent / Agent 停用 / 消息为空。
- 流式过程错误：后端 `onError` 转成错误事件。

> 📌 **不存记录铁律**（后端 [10 §六](../10-在线-Agent-运行环境.md)）：后端无 message 表，前端负责在内存中维持对话历史，**刷新/关闭即清**。前端**禁止**把对话写 localStorage。

## 二、`useChat` 封装（`src/features/chat/use-agent-chat.ts`）

```ts
import { useChat as useAiChat } from '@ai-sdk/react'
import { authStore } from '@/lib/auth-store'
import { useQueryClient } from '@tanstack/react-query'
import { queryKeys } from '@/lib/query-keys'
import { toast } from 'sonner'

export function useAgentChat() {
  const qc = useQueryClient()
  const token = authStore.getState().token

  return useAiChat({
    api: '/api/chat',                          // 走 vite proxy / 生产同源
    headers: token ? { Authorization: `Bearer ${token}` } : {},

    // 流结束后：刷新当前用户（扣费后 credits 变化）
    onFinish: () => {
      qc.invalidateQueries({ queryKey: queryKeys.me })
    },

    // 流开始前的 HTTP 错误（401/402/400）与流过程错误
    onError: (error) => {
      // useChat 的 error 通常是 Error 包装；按 message 判断或尝试解析
      const msg = error.message ?? ''
      if (msg.includes('402') || msg.includes('额度')) {
        toast.error('额度不足，请联系管理员充值')
      } else if (msg.includes('请先创建 Agent')) {
        toast.error('请先到「我的 Agent」创建 Agent 再对话')
      } else {
        toast.error('对话出错，请重试')
      }
    },
  })
}
```

> 💡 `useChat` 把对话 state（messages、input、isSending 等）封装在内部，组件直接解构使用，无需自建 reducer。

## 三、对话页面（`pages/chat.page.tsx`）

```
┌──────────────────────────────────────────┐
│ 与我的 Agent 对话            额度：5000    │  顶栏（credits 实时）
├──────────────────────────────────────────┤
│                                          │
│   [user] 用 list_posts 看看有几篇文章     │
│                                          │
│   [agent] 好的，正在查询…                 │
│          ╭ 工具调用：list_posts ─╮        │   ← 工具调用展示
│          ╰ 返回 5 篇文章 ────────╯        │
│          目前共有 5 篇已发布文章：          │
│          1. xxx (slug: ...)               │
│          ...                              │   ← 流式逐字渲染
│                                          │
├──────────────────────────────────────────┤
│ [输入框________________________] [发送]   │
└──────────────────────────────────────────┘
```

```tsx
// pages/chat.page.tsx（节选）
import { useAgentChat } from '@/features/chat/use-agent-chat'
import { ChatMessages } from '@/features/chat/chat-messages'
import { ChatInput } from '@/features/chat/chat-input'
import { useAuth } from '@/features/auth/use-auth'

export default function ChatPage() {
  const { messages, input, handleInputChange, handleSubmit, status, error, reload } = useAgentChat()
  const { user } = useAuth()

  // 无 Agent 的引导（后端会返回 400，这里提前 UX）
  // 实际是否有 Agent 由 Agent 页管理；此处依赖后端 400 兜底

  return (
    <div className="flex flex-col h-[calc(100vh-4rem)]">
      <div className="border-b px-4 py-2 flex items-center justify-between">
        <span className="font-medium">与我的 Agent 对话</span>
        <span className="text-sm text-muted-foreground">额度：{user?.credits ?? '-'}</span>
      </div>

      <ChatMessages messages={messages} status={status} error={error} onRetry={reload} />

      <ChatInput
        input={input}
        onChange={handleInputChange}
        onSubmit={handleSubmit}
        disabled={status !== 'ready'}
      />
    </div>
  )
}
```

### 3.1 消息渲染（`features/chat/chat-messages.tsx`）

```tsx
import type { UIMessage } from '@ai-sdk/react'
import { Markdown } from '@/components/markdown'
import { cn } from '@/lib/cn'

export function ChatMessages({
  messages, status, error, onRetry,
}: {
  messages: UIMessage[]
  status: string
  error: Error | null
  onRetry: () => void
}) {
  return (
    <div className="flex-1 overflow-y-auto px-4 py-6 space-y-6">
      {messages.length === 0 && (
        <Empty title="开始和你的 Agent 对话" description="对话不会被保存，刷新即清空" />
      )}

      {messages.map((m) => (
        <div key={m.id} className={cn('flex', m.role === 'user' ? 'justify-end' : 'justify-start')}>
          <div className={cn(
            'max-w-[80%] rounded-lg px-4 py-2',
            m.role === 'user' ? 'bg-primary text-primary-foreground' : 'bg-muted',
          )}>
            {/* AI SDK v5 的 message.parts 结构 */}
            {m.parts.map((part, i) => {
              if (part.type === 'text') return <Markdown key={i} content={part.text} />
              if (part.type === 'tool-<name>') return <ToolCallView key={i} part={part} />
              // 其他 part 类型按需
              return null
            })}
          </div>
        </div>
      ))}

      {/* 流式中 */}
      {status === 'streaming' && <TypingIndicator />}

      {/* 错误重试 */}
      {error && (
        <div className="text-center">
          <span className="text-destructive text-sm">对话出错</span>{' '}
          <Button variant="link" onClick={onRetry}>重试</Button>
        </div>
      )}
    </div>
  )
}
```

> 💡 AI SDK v5 用 `message.parts`（text / tool-call / tool-result / reasoning 等）替代 v3 的单一 `content` 字符串。按 part 类型分别渲染。

### 3.2 工具调用展示（`ToolCallView`）

后端注入了 5 个文章工具（后端 [10 §四](../10-在线-Agent-运行环境.md) + `ai/tools.ts`）。Agent 调用工具时，前端展示调用过程让用户感知：

```tsx
function ToolCallView({ part }: { part: { toolName: string; input?: unknown; output?: unknown } }) {
  return (
    <div className="my-2 border rounded p-2 text-xs bg-background">
      <div className="flex items-center gap-1 text-muted-foreground">
        <Wrench className="size-3" />
        调用工具：{part.toolName}
        {!part.output && <span className="animate-pulse">…</span>}
      </div>
      {part.output && (
        <pre className="mt-1 max-h-40 overflow-auto">{JSON.stringify(part.output, null, 2)}</pre>
      )}
    </div>
  )
}
```

> 💡 工具结果可折叠/摘要展示，避免刷屏。`list_posts` 的结果可渲染成文章卡片列表而非裸 JSON（体验更好）。

## 四、流式鉴权错误的处理

> ⚠️ `useChat` 的 fetch 不走 `request()`（见 [03 §八](./03-API-层与请求封装.md)），所以 401 不会自动跳登录。需在 `useAgentChat` 的 `onError` 补一刀。

AI SDK 的 `onError` 收到的 error 通常是 `Error`，其 message 可能含状态码信息。更稳妥的做法是用 `fetch` 包装拦截，或检查 `error.cause`：

```ts
onError: (error) => {
  // 部分版本会把 Response 挂在 error 上
  const status = (error as any).status ?? (error as any).response?.status
  if (status === 401) {
    authStore.getState().clear()
    qc.clear()
    router.navigate('/login', { state: { from: router.state.location } })
    return
  }
  // 其余按 §二处理
}
```

> 💡 不同 AI SDK 版本错误结构略有差异，联调时实测 `console.error(error)` 确认字段，再精确判断。

## 五、刷新即清空（不存记录的落点）

`useChat` 的 `messages` 是组件 state，离开页面/刷新即销毁——天然符合需求。需确保：

- **不**把 `messages` 持久化到 localStorage/sessionStorage。
- **不**在路由切换时保留对话（用 `useAgentChat` 的实例作用域，离开 `/chat` 即销毁）。
- 页面顶部明确提示「对话不保存，刷新即清空」，避免用户误解。

```tsx
// 顶部提示条
<div className="bg-amber-50 text-amber-800 text-xs px-4 py-1 text-center">
  对话仅在本页有效，刷新或离开后清空
</div>
```

> 📌 与后端 [10 §六](../10-在线-Agent-运行环境.md)「完全不建 message 表」呼应——前后端都不存，隐私闭环。

## 六、余额预检与提示

后端预检 `credits < 1` 返回 402（后端 [10 §五](../10-在线-Agent-运行环境.md)）。前端可做更早的预检：

```tsx
const { user } = useAuth()
const canChat = (user?.credits ?? 0) > 0

<ChatInput
  disabled={!canChat || status !== 'ready'}
  placeholder={canChat ? '输入消息…' : '额度不足，无法对话'}
/>
{!canChat && (
  <p className="text-center text-sm text-destructive">
    额度不足，请联系管理员充值
  </p>
)}
```

> 💡 前端预检只是 UX 优化，后端 402 是最终保障。对话扣费按实际 Token（可能一次扣多个 credit），所以 `credits > 0` 但不足以支撑完整对话时仍可能中途 402（后端 [10 §七](../10-在线-Agent-运行环境.md) 宽松策略）。

## 七、System Prompt 与 Agent 依赖

- 对话依赖用户已创建 Agent（后端 [10 §五](../10-在线-Agent-运行环境.md)：无 Agent 返回 400）。
- System Prompt 在后端从用户的 Agent 注入（后端 [10 §九](../10-在线-Agent-运行环境.md)），**前端无需也不应**感知 prompt 内容。
- 前端可在对话页顶部展示当前 Agent 名称/头像（`useMineAgent()`），让用户知道在和哪个 Agent 对话。

> 💡 安全：后端 [10 §九](../10-在线-Agent-运行环境.md) 明确"不要把 system prompt 暴露给对话流"——前端也不要尝试展示或编辑它（编辑在 Agent 管理页）。

## 八、验收用例（对应后端 TC-004）

| 用例 | 操作 | 预期 |
|------|------|------|
| TC-004 流式对话 | 发送消息 | 逐字流式渲染，结束扣费、credits 刷新 |
| 刷新页面 | 对话中刷新 | 对话清空（无记录） |
| 无 Agent 对话 | 未创建 Agent 发消息 | 400 → toast「请先创建 Agent」 |

| 工具调用 | 让 Agent 调 list_posts | 工具调用过程展示 + 结果回到流 |
| 401 处理 | token 失效后发消息 | 自动跳登录 |
