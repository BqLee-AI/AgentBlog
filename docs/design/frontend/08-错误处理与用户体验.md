# 08 · 错误处理与用户体验

> 本篇定义前端的**全局错误兜底**（ErrorBoundary）、**错误码 → toast/跳转的映射落地**、**加载/空态/网络异常**等通用 UX。
> 对接后端 [12 · 统一错误处理与响应规范](../12-统一错误处理与响应规范.md) 的错误码表。

---

## 一、错误处理的三层防线

| 层 | 处理对象 | 工具 | 例子 |
|----|----------|------|------|
| **组件内** | 预期的、需就地展示的错误 | try/catch + RHF setError | 登录失败、表单字段错误回填 |
| **Query 级** | 数据请求的默认错误展示 | `useQuery` 的 `error` + 局部 ErrorState | 列表加载失败显示重试 |
| **全局兜底** | 未捕获异常、全局副作用 | ErrorBoundary + 全局 toast | 401 自动跳登录、402 全局提示、渲染崩溃 |

> 📌 **原则**：默认错误走全局（toast/跳转），只有需要"覆盖默认行为"才局部 catch。这样错误处理一致、不漏。

---

## 二、全局 ErrorBoundary（`src/app/error-boundary.tsx`）

捕获**渲染期**异常（组件 throw、render crash），避免白屏。

```tsx
import { Component, type ReactNode } from 'react'

interface State { error: Error | null }

export class ErrorBoundary extends Component<{ children: ReactNode }, State> {
  state: State = { error: null }

  static getDerivedStateFromError(error: Error): State {
    return { error }
  }

  componentDidCatch(error: Error, info: React.ErrorInfo) {
    // 上报到日志（v1 console 即可）
    console.error('💥 渲染错误:', error, info)
  }

  render() {
    if (this.state.error) {
      return (
        <div className="flex flex-col items-center justify-center min-h-screen gap-4 p-8">
          <h1 className="text-2xl font-bold">页面出错了</h1>
          <p className="text-muted-foreground">抱歉，发生了一些错误，请刷新或返回首页。</p>
          <div className="flex gap-2">
            <Button variant="outline" onClick={() => window.location.reload()}>刷新页面</Button>
            <Button onClick={() => { this.setState({ error: null }); window.location.href = '/' }}>
              返回首页
            </Button>
          </div>
        </div>
      )
    }
    return this.props.children
  }
}
```

> 💡 ErrorBoundary 挂在 `RootProviders` 最外层（见 [01 §四](./01-工程化与目录结构.md)）。Async 错误（如 onClick 内 await 的 rejection）不会被 ErrorBoundary 捕获——那部分由全局 toast 兜底（见 §四）。

## 三、错误码 → 处理动作映射表（落地版）

后端 [12 §三](../12-统一错误处理与响应规范.md) 错误码表的前端处理：

| code | 默认动作 | 副作用 | 是否可被组件覆盖 |
|------|----------|--------|------------------|
| `VALIDATION_ERROR` | 字段错误回填表单（如有 details） | — | ✅ 表单内联 |
| `BAD_REQUEST` | toast `error.message` | — | ✅ |
| `UNAUTHORIZED` | **自动**清登录态 + 跳登录 + 清缓存 | request 层统一处理 | ❌（全局） |
| `FORBIDDEN` | toast `error.message`（如「无权修改他人文章」） | — | ✅ 跳 `/forbidden` |
| `NOT_FOUND` | toast 或跳 404 | — | ✅ 阅读端跳 404 |
| `INSUFFICIENT_CREDITS` | **自动**toast「额度不足，请联系管理员充值」 | CustomEvent | ✅ 对话页预检 |
| `CONFLICT` | toast `error.message`（如「每个用户只能创建 1 个 Agent」） | — | ✅ |
| `INTERNAL_ERROR` | toast「服务器内部错误，请稍后重试」 | — | — |
| 网络错误 | toast「网络异常，请检查连接」 | — | ✅ 重试按钮 |

> 💡 **message 直接用后端的中文**（后端 [12 §三](../12-统一错误处理与响应规范.md) 全部中文）。前端不二次翻译，避免文案漂移。只有极少数"前端需改写"的场景（如 402 统一加充值引导）才覆盖。

## 四、全局 Toast 系统

用 shadcn 的 `useToast` 或 `sonner`（更现代、API 简洁）。本套文档推荐 sonner。

### 4.1 挂载（`RootProviders`）

```tsx
import { Toaster } from 'sonner'
// ...
<Toaster position="top-center" richColors closeButton />
```

### 4.2 Query 全局错误默认展示（`query-client.ts`）

```ts
import { toast } from 'sonner'
import { ApiError } from '@/lib/http-error'

export const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      // ... 见 04
      retry: (count, error) => {
        if (error instanceof ApiError && [400, 401, 402, 403, 404, 409].includes(error.status)) {
          return false
        }
        return count < 2
      },
    },
  },
  // 全局 mutation 错误：未被局部 catch 的都 toast
})

// 全局 mutation 错误兜底（在挂载 QueryClientProvider 时配置）
queryClient.setMutationDefaults(['/api'], {
  onError: (error) => showErrorToast(error),
})

export function showErrorToast(error: unknown) {
  if (error instanceof ApiError) {
    // 已被 request 层处理的（401 跳转、402 事件）不重复 toast
    if (error.isUnauthorized) return
    toast.error(error.message)
  } else {
    toast.error('网络异常，请检查连接')
  }
}
```

> 💡 TanStack Query 的 mutation 默认错误传播给 `onError`（hook 层）。若 hook 未定义 onError，错误会冒泡到全局。配合 `showErrorToast` 兜底，业务层无需每个 mutation 都写错误处理。

### 4.3 402 全局监听（监听 §三的 CustomEvent）

```tsx
// src/app/root-providers.tsx
import { useEffect } from 'react'
import { toast } from 'sonner'

useEffect(() => {
  const handler = () => toast.error('额度不足，请联系管理员充值')
  window.addEventListener('app:insufficient-credits', handler)
  return () => window.removeEventListener('app:insufficient-credits', handler)
}, [])
```

> 💡 为什么 402 走事件而非直接在 request 层 toast？见 [03 §五](./03-API-层与请求封装.md)——request 是纯逻辑层，不依赖 UI。事件解耦后，测试时也能 mock。

## 五、加载态（Loading）

| 场景 | 展示 |
|------|------|
| 首次查询（无缓存） | `<ListSkeleton />` / `<PageSkeleton />`（灰块） |
| 翻页/筛选（有缓存） | `placeholderData: prev` 保旧数据，顶部细进度条 |
| 表单提交 | 按钮 `disabled` + 文案「提交中…」 |
| 路由切换 | `<Suspense fallback={<PageSkeleton />}>` |
| 全屏校验（AuthProvider） | `<Spin fullscreen>` |

```tsx
// 通用列表加载
if (isPending) return <ListSkeleton rows={5} />
if (error) return <ErrorState message={error.message} onRetry={() => refetch()} />
if (!data?.items.length) return <Empty title="暂无文章" action={<Button>新建</Button>} />
return <PostList items={data.items} />
```

> 💡 顺序：pending → error → empty → 内容。避免把"空"和"加载中"混淆。

## 六、空态（Empty）

每个列表/详情页都应有友好的空态，而非空白。区分"真没数据"和"加载中"：

- 真没数据：`<Empty title="暂无文章" description="点击新建创建第一篇" action={...} />`
- 标签无文章：`<Empty title="该标签下还没有文章" />`
- Credits 流水为空：`<Empty title="暂无额度记录" />`

## 七、网络异常与重试

```tsx
// network error 时提供重试
if (error && !(error instanceof ApiError)) {
  return <ErrorState message="网络异常，请检查连接" onRetry={() => refetch()} />
}
```

TanStack Query 已对 5xx/网络自动重试 2 次（见 [04 §五](./04-状态管理与数据获取.md)），失败后给用户手动重试入口。

## 八、401 的完整闭环

```
任意请求 → 后端返回 401 UNAUTHORIZED
   ↓
request() 的 handleSideEffects 捕获
   ↓
authStore.clear()         （清 token + user）
queryClient.clear()       （清所有缓存，防残留）
router.navigate('/login', state: { from })   （回跳来源）
   ↓
登录页登录成功 → setAuth(token, user) → navigate(from)
```

> 💡 这个闭环由 `request()` 统一处理（见 [03 §五](./03-API-层与请求封装.md)），业务层无感。流式 `/api/chat` 的 401 需在 `useAgentChat` 的 onError 补一刀（见 [07 §四](./07-在线对话与流式实现.md)）。

## 九、文案规范

- **错误文案用后端 message**：后端 [12 §三](../12-统一错误处理与响应规范.md) 全部中文，前端直接展示。如「无权修改他人文章」「每个用户只能创建 1 个 Agent」。
- **前端补充的引导文案**：简洁、动作明确。如「额度不足，请联系管理员充值」「请先到「我的 Agent」创建 Agent 再对话」。
- **按钮文案**：动词开头。如「新建文章」「保存」「删除」「重试」。
- **提示色**：错误用 destructive（红），警告用 amber（黄），成功用 primary/green。

## 十、无障碍（a11y）基础

v1 投入产出比最高的几项：

- 所有交互元素（按钮、链接）有清晰的 `aria-label`（纯图标按钮必须）。
- 表单 `<label>` 与 `<input>` 关联（shadcn Form 已处理）。
- 错误提示用 `role="alert"`（toast 自动）。
- 颜色对比度满足 WCAG AA（shadcn 默认 token 已达标）。
- 键盘可达：所有交互能用 Tab 聚焦、Enter 触发（Radix 基座已处理）。
