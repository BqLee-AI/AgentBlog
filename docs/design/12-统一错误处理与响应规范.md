# 12 · 统一错误处理与响应规范

> 本篇定义**全系统对外响应的统一结构**、错误码体系、中文提示规范。
> 对应需求 §5「易用性：错误提示使用中文」与「机器可读性」。
> 贯穿所有模块。

---

## 一、统一响应结构

所有 REST 接口（`/api/*`）必须返回：

### 成功

```jsonc
{
  "ok": true,
  "data": <any>,          // 业务数据
  "meta"?: { ... }         // 可选：分页、额外信息
}
```

### 失败

```jsonc
{
  "ok": false,
  "error": {
    "code": "NOT_FOUND",            // 机器可读的错误码（大写蛇形）
    "message": "文章不存在",          // 人类可读，中文（需求要求）
    "details"?: { ... }              // 可选：字段级错误等
  }
}
```

> 📌 `ok` 是布尔而非 HTTP 状态码——便于 Agent 客户端（MCP）统一判断。
> 📌 `code` 与 `message` 分离：Agent 读 `code` 做逻辑，人读 `message`。

## 二、HTTP 状态码使用约定

| 状态 | 何时用 |
|------|--------|
| 200 | 读取/更新成功 |
| 201 | 创建成功 |
| 204 | 删除成功（可选，本系统返回 200 + `{ deleted: true }`） |
| 400 | 请求参数错误（Zod 校验失败、业务规则不满足） |
| 401 | 未认证（无 token / token 无效） |
| 403 | 无权限（角色不足、资源非自己） |
| 404 | 资源不存在 |
| 402 | 📌 额度不足（需求指定） |
| 409 | 冲突（用户名已存在、已建过 Agent） |
| 500 | 服务端未知错误 |

## 三、错误码表

| code | HTTP | 含义 | 中文提示 |
|------|------|------|----------|
| `VALIDATION_ERROR` | 400 | 参数校验失败 | （字段级，见 details） |
| `BAD_REQUEST` | 400 | 通用业务规则不满足 | （动态） |
| `UNAUTHORIZED` | 401 | 未登录/token 失效 | 未提供认证信息 / 登录已过期，请重新登录 |
| `FORBIDDEN` | 403 | 角色或资源权限不足 | 无权执行此操作 / 无权修改他人文章 |
| `NOT_FOUND` | 404 | 资源不存在 | 文章不存在 / Agent 不存在 |
| `INSUFFICIENT_CREDITS` | 402 | 📌 额度不足 | 额度不足，请联系管理员充值 |
| `CONFLICT` | 409 | 资源冲突 | 用户名已存在 / 每个用户只能创建 1 个 Agent |
| `INTERNAL_ERROR` | 500 | 未知错误 | 服务器内部错误，请稍后重试 |

## 四、`HttpError` 类（`src/lib/errors.ts`）

```ts
export class HttpError extends Error {
  constructor(
    readonly status: number,
    readonly code: string,
    message: string,
    readonly details?: Record<string, unknown>,
  ) {
    super(message)
    this.name = 'HttpError'
  }

  // 常用静态工厂
  static badRequest(msg = '请求参数错误', details?: Record<string, unknown>) {
    return new HttpError(400, 'BAD_REQUEST', msg, details)
  }
  static unauthorized(msg = '未提供认证信息') { return new HttpError(401, 'UNAUTHORIZED', msg) }
  static forbidden(msg = '无权执行此操作') { return new HttpError(403, 'FORBIDDEN', msg) }
  static notFound(msg = '资源不存在') { return new HttpError(404, 'NOT_FOUND', msg) }
  static payment(msg = '额度不足') { return new HttpError(402, 'INSUFFICIENT_CREDITS', msg) }
  static conflict(msg = '资源冲突') { return new HttpError(409, 'CONFLICT', msg) }
  static internal(msg = '服务器内部错误') { return new HttpError(500, 'INTERNAL_ERROR', msg) }
}
```

## 五、响应助手（`src/lib/response.ts`）

```ts
import type { Context } from 'hono'

export function ok<T>(c: Context, data: T, status = 200) {
  return c.json({ ok: true, data }, status)
}

export function okWithMeta<T>(c: Context, data: T, meta: Record<string, unknown>, status = 200) {
  return c.json({ ok: true, data, meta }, status)
}

export function fail(c: Context, status: number, code: string, message: string, details?: Record<string, unknown>) {
  return c.json({ ok: false, error: { code, message, ...(details ? { details } : {}) } }, status)
}
```

## 六、全局错误处理（`src/middlewares/error-handler.ts`）

> 📌 这是统一响应的兜底。service 抛 `HttpError`，路由层不 catch，全部由此中间件转 JSON。

```ts
import type { ErrorHandler } from 'hono'
import { ZodError } from 'zod'
import { HttpError } from '@/lib/errors'
import { env } from '@/config/env'

export const errorHandler: ErrorHandler = (err, c) => {
  // 1. Zod 校验错误 → 400 + 字段级 details
  if (err instanceof ZodError) {
    return c.json({
      ok: false,
      error: {
        code: 'VALIDATION_ERROR',
        message: '请求参数校验失败',
        details: err.flatten().fieldErrors,
      },
    }, 400)
  }

  // 2. 业务错误 → 按其定义返回
  if (err instanceof HttpError) {
    return c.json({
      ok: false,
      error: {
        code: err.code,
        message: err.message,
        ...(err.details ? { details: err.details } : {}),
      },
    }, err.status)
  }

  // 3. 未知错误 → 500，生产环境不泄露堆栈
  console.error('💥 未处理错误:', err)
  const message = env.NODE_ENV === 'production'
    ? '服务器内部错误，请稍后重试'
    : err.message
  return c.json({
    ok: false,
    error: { code: 'INTERNAL_ERROR', message },
  }, 500)
}
```

在 `app.ts` 注册：

```ts
app.onError(errorHandler)

// 兜底：404 路由
app.notFound((c) => c.json({ ok: false, error: { code: 'NOT_FOUND', message: '接口不存在' } }, 404))
```

## 七、Zod 校验错误的对接

Hono 用 `@hono/zod-validator`，校验失败默认返回 400。但默认响应格式不符本系统，需自定义 hook：

```ts
import { zValidator } from '@hono/zod-validator'
import { ZodError } from 'zod'

// 自定义 hook 把 zod 错误转成统一格式
export const zodCheck = (target, schema) =>
  zValidator(target, schema, (result, c) => {
    if (!result.success) {
      const err = result.error
      return c.json({
        ok: false,
        error: {
          code: 'VALIDATION_ERROR',
          message: '请求参数校验失败',
          details: err.flatten().fieldErrors,
        },
      }, 400)
    }
  })

// 使用
posts.post('/', zodCheck('json', createPostSchema), async (c) => { /* ... */ })
```

## 八、流式接口的错误处理（`/api/chat`）

流式接口（SSE）不能在流开始后返回 HTTP 错误码。策略：**流开始前的错误**（无 Agent、余额不足、参数错）正常返回 HTTP 错误；**流过程中的错误**通过 `onError` 转成错误事件。

```ts
// ai/runtime.ts
return streamText({
  // ...
  onError: ({ error }) => {
    console.error('流式错误:', error)
    return '对话发生错误，请重试'
  },
})
```

> 💡 鉴权与预检（余额、Agent 存在性）都在 `streamText` 之前做，确保能正常返回 HTTP 401/402。

## 九、分页响应规范

列表接口统一返回分页元信息：

```jsonc
{
  "ok": true,
  "data": [ /* 文章数组 */ ],
  "meta": {
    "page": 1,
    "pageSize": 10,
    "total": 42
  }
}
```

或把分页包进 data：

```jsonc
{ "ok": true, "data": { "items": [...], "page": 1, "pageSize": 10, "total": 42 } }
```

> 💡 本套文档统一用第二种（`data.items` + `data.total`），与 06 的 `list` 返回结构一致。
