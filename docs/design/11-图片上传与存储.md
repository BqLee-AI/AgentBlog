# 11 · 图片上传与存储

> 本篇实现需求 §2.6「对象/本地存储」：文章封面图、用户与 Agent 头像上传，存 URL 路径于数据库。
> 对应任务 9「图片上传与存储」。
>
> **强约束**：上传后存 URL 路径于数据库（需求 §2.6）；限制大小与类型（需求 §4.2）。

---

## 一、设计要点

- 用 Hono 原生 `c.req.parseBody()` 解析 multipart/form-data（无需 multer 类中间件）。
- **本地存储**（v1 默认）+ **可替换的对象存储抽象**（后续扩展）。
- 文件名用 hash/uuid 防冲突与路径遍历。
- 限制大小（`UPLOAD_MAX_SIZE_MB`，默认 5MB）与 MIME（仅图片）。
- 上传成功返回可访问 URL，落库的是 URL 字符串。

## 二、存储抽象（`src/lib/storage.ts`）

> 💡 定义统一接口，本地与未来 S3/OSS 可互换。

```ts
import { mkdir, writeFile } from 'node:fs/promises'
import { join, extname } from 'node:path'
import { randomUUID } from 'node:crypto'
import { env } from '@/config/env'

export interface StorageResult {
  url: string       // 可访问的 URL（落库用）
  key: string       // 存储内部 key（删除用）
  size: number
}

export interface Storage {
  save(file: File, dir: string): Promise<StorageResult>
  delete(key: string): Promise<void>
}

// ---------- 本地实现 ----------
class LocalStorage implements Storage {
  constructor(private baseUrl = '/uploads') {}

  async save(file: File, dir: string): Promise<StorageResult> {
    const ext = extname(file.name).toLowerCase()
    const filename = `${randomUUID()}${ext}`
    const relPath = `${dir}/${filename}`
    const absPath = join(env.UPLOAD_DIR, dir, filename)

    await mkdir(join(env.UPLOAD_DIR, dir), { recursive: true })
    const buf = Buffer.from(await file.arrayBuffer())
    await writeFile(absPath, buf)

    return { url: `${this.baseUrl}/${relPath}`, key: relPath, size: file.size }
  }

  async delete(key: string) {
    const { unlink } = await import('node:fs/promises')
    await unlink(join(env.UPLOAD_DIR, key)).catch(() => {})
  }
}

export const storage: Storage = new LocalStorage()
// 未来：export const storage: Storage = new S3Storage({...})
```

## 三、校验（`src/lib/upload-validate.ts`）

```ts
import { env } from '@/config/env'
import { HttpError } from '@/lib/errors'

const ALLOWED = new Set(['image/jpeg', 'image/png', 'image/webp', 'image/gif'])

export function validateImage(file: File) {
  if (!ALLOWED.has(file.type)) {
    throw HttpError.badRequest(`不支持的图片类型：${file.type}，仅支持 jpg/png/webp/gif`)
  }
  const maxBytes = env.UPLOAD_MAX_SIZE_MB * 1024 * 1024
  if (file.size > maxBytes) {
    throw HttpError.badRequest(`图片不能超过 ${env.UPLOAD_MAX_SIZE_MB}MB`)
  }
}
```

## 四、静态文件托管

上传的文件需要通过 HTTP 可访问。在 `app.ts` 挂载静态目录：

```ts
import { serveStatic } from 'hono/bun'

// 静态文件：/uploads/* → data/uploads/
app.use('/uploads/*', serveStatic({ root: './' , rewriteRequestPath: (p) => p }))
```

> 💡 `hono/bun` 的 `serveStatic` 从相对路径读文件。需确保 `UPLOAD_DIR=./data/uploads` 与挂载路径对齐。
> 💡 生产环境建议改用 Nginx/CDN 直接服务静态文件，后端只负责写入。

## 五、上传路由（`src/routes/upload.routes.ts`）

```ts
import { Hono } from 'hono'
import { authMiddleware } from '@/middlewares/auth'
import { storage } from '@/lib/storage'
import { validateImage } from '@/lib/upload-validate'
import { HttpError } from '@/lib/errors'
import { ok } from '@/lib/response'

export const upload = new Hono()

upload.use('*', authMiddleware)

upload.post('/', async (c) => {
  const body = await c.req.parseBody()
  const file = body.file
  if (!(file instanceof File)) throw HttpError.badRequest('请上传 file 字段')

  // 限制用途（可选）：按 form field 区分头像/封面
  const purpose = (body.purpose as string) || 'cover'
  const dir = ['avatar', 'cover'].includes(purpose) ? purpose : 'misc'

  validateImage(file)
  const result = await storage.save(file, dir)

  return ok(c, result, 201)
})
```

## 六、接口契约

### `POST /api/upload`

```
Content-Type: multipart/form-data
Authorization: Bearer <token>

字段:
  file: <二进制>      (必填)
  purpose: avatar|cover|misc  (可选，默认 cover)
```

```jsonc
// res 201
{
  "ok": true,
  "data": {
    "url": "/uploads/cover/3f9a-uuid.png",
    "key": "cover/3f9a-uuid.png",
    "size": 234567
  }
}
```

前端拿到 `url` 后，再调文章/用户/Agent 的更新接口把 URL 写进 `coverUrl` / `avatarUrl`。

## 七、使用流程（前端视角）

```
1. 用户选图 → POST /api/upload
2. 拿到 { url: "/uploads/cover/xxx.png" }
3. PATCH /api/posts/:id { coverUrl: url }   或
   PATCH /api/agents/:id { avatarUrl: url }
```

## 八、验收用例

| 用例 | 操作 | 预期 |
|------|------|------|
| 上传 PNG | POST file | 201，返回可访问 url |
| 上传 10MB | 超限 | 400「不能超过 5MB」 |
| 上传 .txt | 非图片类型 | 400「不支持的图片类型」 |
| 访问上传图 | GET /uploads/.. | 200 图片 |
| 未登录上传 | 无 token | 401 |

## 九、安全注意

- **路径遍历**：文件名用 `randomUUID()`，**不**用用户原始文件名。
- **MIME 欺骗**：v1 信任 `file.type`；高安全场景可读 magic number 校验。
- **未鉴权访问**：本实现 `/uploads/*` 公开可读（封面/头像本就公开）。如需私有，改用签名 URL。
- **清理**：被替换/删除的旧文件可加定期清理任务（v1 不做）。
