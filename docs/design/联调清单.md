## 三、在线 Agent 流式联调（手工，doc 13 §六）

```bash
curl -N -X POST http://localhost:3000/api/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"用 list_posts 看看有几篇文章"}]}'
```

- [ ] 🟡 流式输出（逐块到达，非一次性返回）

- [ ] 🟡 流结束后 credit_log 多一条（type=agent_token）

## 五、部署验证（doc 14 §十）

```bash
# 1. 配置生产环境变量
cp apps/api/.env.example apps/api/.env.production
vim apps/api/.env.production   # 改 JWT_SECRET / 超管密码 / OPENAI_API_KEY

# 2. 启动（自动 migrate，不含 seed）
docker compose up -d

# 3. 健康检查
curl http://localhost:3000/health   # → { ok:true, data:{status:'running'} }

# 4. 首次部署：手动 seed
docker compose exec api bun run db:seed   # 创建超管 + 示例标签

# 5. 超管登录验证
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"<你的超管密码>"}'
```

- [ ] 🟡 `docker compose up` → `/health` 200
- [ ] 🟡 容器重启后表不丢失（migrate 自动跑，数据卷持久化）
- [ ] 🟡 `db:seed` 后超管可登录
- [ ] 🟡 🔴 生产不会自动 seed（防 .env 密码变更后意外重置超管）
