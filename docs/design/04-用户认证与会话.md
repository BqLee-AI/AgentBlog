# 04 · 用户认证与会话

> 本篇实现需求 §4.1「用户登录」：用户名 + 密码注册登录、密码加密存储、JWT 会话。
> 对应任务 5「用户注册登录与会话管理」。
>
> **强约束**：用户名唯一且长度 3–32；密码加密存储（需求原文）。

---

## 一、模块概览

| 能力 | 接口 | 鉴权 |
|------|------|------|
| 注册（超管/管理员创建用户用） | `POST /api/auth/register` | 仅 admin/super_admin |
| 登录 | `POST /api/auth/login` | 公开 |
| 获取当前用户（含 credits） | `GET /api/auth/me` | JWT |
| 登出 | `POST /api/auth/logout` | JWT |

> 💡 需求 §2.3 描述用户由「管理员建用户」产生。本系统**不开放自助注册**，普通用户由 admin 创建。超管通过 seed 生成（见 03）。

## 二、密码哈希（`src/lib/hash.ts`）

> 📌 需求要求「加密存储」。用 bcrypt 加盐，cost=10 足够。

```ts
import bcrypt from 'bcryptjs'

const COST = 10

export async function hashPassword(plain: string): Promise<string> {
  return bcrypt.hash(plain, COST)
}

export async function verifyPassword(plain: string, hash: string): Promise<boolean> {
  return bcrypt.compare(plain, hash)
}
```

## 三、JWT 工具（`src/lib/jwt.ts`）

Hono 自带 JWT，但为了 payload 类型清晰，封装一层：

```ts
import { sign, verify, decode } from 'hono/utils/jwt/jws'
import { env } from '@/config/env'
import { HttpError } from '@/lib/errors'

export interface JwtPayload {
  sub: number       // user.id
  role: string      // 'super_admin' | 'admin' | 'user'
  iat?: number
  exp?: number
}

export async function signToken(payload: Omit<JwtPayload, 'iat' | 'exp'>): Promise<string> {
  return sign(payload, env.JWT_SECRET)
}

export async function verifyToken(token: string): Promise<JwtPayload> {
  try {
    const payload = (await verify(token, env.JWT_SECRET)) as JwtPayload
    return payload
  } catch {
    throw HttpError.unauthorized('登录已过期，请重新登录')
  }
}
```

> 💡 `hono/utils/jwt` 的 `sign/verify` 用 HS256，对称密钥适合单服务。过期时间在 `.env` 的 `JWT_EXPIRES_IN`（注意 Hono JWT 不直接支持 `7d` 字符串，需转成秒数——可在 signToken 内转）。

## 四、会话载体：Authorization Header

> 💡 博客系统多 SPA，用 `Authorization: Bearer <token>` 最简单；如需 Cookie，可改用 `setCookie`（注意 CSRF）。本套文档采用 **Bearer Header**。

## 五、Schema（`src/modules/auth/auth.schema.ts`）

```ts
import { z } from 'zod'

export const loginSchema = z.object({
  username: z.string().min(3).max(32),
  password: z.string().min(6, '密码至少 6 位'),
})
export type LoginDTO = z.infer<typeof loginSchema>

export const createUserSchema = z.object({
  username: z.string().min(3).max(32),
  password: z.string().min(6),
  role: z.enum(['admin', 'user']).default('user'),  // 不允许直接建超管
  credits: z.number().int().min(0).default(0),
})
export type CreateUserDTO = z.infer<typeof createUserSchema>
```

## 六、Service（`src/modules/auth/auth.service.ts`）

```ts
import { eq } from 'drizzle-orm'
import { db } from '@/db/client'
import { users } from '@/db/schema'
import { hashPassword, verifyPassword } from '@/lib/hash'
import { signToken } from '@/lib/jwt'
import { HttpError } from '@/lib/errors'
import type { LoginDTO, CreateUserDTO } from './auth.schema'

export const authService = {
  async login(dto: LoginDTO) {
    const [user] = await db.select().from(users).where(eq(users.username, dto.username)).limit(1)
    if (!user) throw HttpError.unauthorized('用户名或密码错误')

    const ok = await verifyPassword(dto.password, user.passwordHash)
    if (!ok) throw HttpError.unauthorized('用户名或密码错误')

    if (user.status === 'disabled') throw HttpError.forbidden('账号已禁用')

    const token = await signToken({ sub: user.id, role: user.role })
    return {
      token,
      user: { id: user.id, username: user.username, role: user.role, credits: user.credits, avatarUrl: user.avatarUrl },
    }
  },

  async createUser(dto: CreateUserDTO, creatorRole: string) {
    if (dto.role === 'super_admin' && creatorRole !== 'super_admin') {
      throw HttpError.forbidden('仅超管可创建超管')
    }
    const [exist] = await db.select().from(users).where(eq(users.username, dto.username)).limit(1)
    if (exist) throw HttpError.conflict('用户名已存在')

    const [user] = await db
      .insert(users)
      .values({
        username: dto.username,
        passwordHash: await hashPassword(dto.password),
        role: dto.role,
        credits: dto.credits,
      })
      .returning()
    return user
  },

  async me(userId: number) {
    const [user] = await db.select().from(users).where(eq(users.id, userId)).limit(1)
    if (!user) throw HttpError.notFound('用户不存在')
    return user
  },
}
```

> 💡 登录失败统一返回「用户名或密码错误」，避免账号枚举攻击。

## 七、鉴权中间件（`src/middlewares/auth.ts`）

> 📌 这是后续所有受保护接口的前置依赖。

```ts
import type { MiddlewareHandler } from 'hono'
import { verifyToken } from '@/lib/jwt'
import { HttpError } from '@/lib/errors'
import { db } from '@/db/client'
import { users } from '@/db/schema'
import { eq } from 'drizzle-orm'

export interface AuthVars {
  user: {
    id: number
    username: string
    role: 'super_admin' | 'admin' | 'user'
    credits: number
  }
}

export const authMiddleware: MiddlewareHandler = async (c, next) => {
  const header = c.req.header('Authorization')
  if (!header?.startsWith('Bearer ')) throw HttpError.unauthorized('未提供认证信息')

  const token = header.slice(7)
  const payload = await verifyToken(token)

  // 实时拉取用户（处理 token 有效但账号被禁用/删除的情况）
  const [user] = await db.select().from(users).where(eq(users.id, payload.sub)).limit(1)
  if (!user || user.status === 'disabled') throw HttpError.unauthorized('账号不可用')

  c.set('user', {
    id: user.id,
    username: user.username,
    role: user.role,
    credits: user.credits,
  })
  await next()
}

// 让 Hono 知道 c.var.user 的类型
declare module 'hono' {
  interface ContextVariableMap extends AuthVars {}
}
```

> 💡 `declare module 'hono'` 给 `c.var.user` 加类型，是 Hono 类型安全的官方做法。

## 八、可选：公开中间件（`maybeAuth`）

阅读页某些场景需要「登录可看更多、未登录看公开」，用可选鉴权：

```ts
export const maybeAuth: MiddlewareHandler = async (c, next) => {
  const header = c.req.header('Authorization')
  if (header?.startsWith('Bearer ')) {
    try {
      const payload = await verifyToken(header.slice(7))
      const [user] = await db.select().from(users).where(eq(users.id, payload.sub)).limit(1)
      if (user?.status === 'active') c.set('user', /* 同上 */)
    } catch { /* 忽略，按未登录处理 */ }
  }
  await next()
}
```

## 九、Routes（`src/routes/auth.routes.ts`）

```ts
import { Hono } from 'hono'
import { zValidator } from '@hono/zod-validator'
import { authService } from '@/modules/auth/auth.service'
import { loginSchema, createUserSchema } from '@/modules/auth/auth.schema'
import { authMiddleware } from '@/middlewares/auth'
import { requireRole } from '@/middlewares/rbac'
import { ok } from '@/lib/response'

export const auth = new Hono()

auth.post('/login', zValidator('json', loginSchema), async (c) => {
  const result = await authService.login(c.req.valid('json'))
  return ok(c, result)
})

// 创建用户（管理员建用户）
auth.post(
  '/register',
  authMiddleware,
  requireRole('admin', 'super_admin'),
  zValidator('json', createUserSchema),
  async (c) => {
    const dto = c.req.valid('json')
    const user = await authService.createUser(dto, c.var.user.role)
    return ok(c, user, 201)
  },
)

auth.get('/me', authMiddleware, async (c) => {
  const user = await authService.me(c.var.user.id)
  return ok(c, user)
})
```
